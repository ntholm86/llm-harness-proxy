Here is the exact anatomy of what intercepting tokens in flight looks like in practice (how something like the `evo` CLI or a production-grade external protocol works):

### 1. The API Boundary (The Choke Point)
You do not use a chat interface. The Harness is typically a backend process (like a Python CLI, a Rust binary, or a Node server) that manages the raw HTTP connection to the LLM provider (OpenAI, Anthropic, etc.). The LLM has zero direct access to the file system, terminal, or its own logs. 

### 2. The Structured Output Mandate
The Harness prompts the LLM with a strict schema requirement. It tells the model: *"You will only respond in this exact JSON structure. If you deviate, your output will be rejected."*
```json
{
  "internal_reasoning": "...",
  "action": {
    "type": "execute_shell",
    "command": "..."
  }
}
```

### 3. The Stream Fork (In-Flight Interception)
This is the most critical part. As the tokens stream back from the API over the network, the Harness intercepts them byte by byte. It **forks** the data stream into two separate physical processes:

*   **Path A (The Immutable Ledger):** The moment the `"internal_reasoning"` string completes, the Harness writes it straight to the hard drive (e.g., `log.md` or a database). **The LLM is completely unaware this is happening.** It is not "calling a tool" to log its thoughts; the environment is simply catching its exhaust and permanently freezing it.
*   **Path B (The Action Queue):** The Harness takes the `"command"` portion of the JSON, holds it in memory, validates it, and prepares to run it.

### 4. Execution and Blind Feedback
The Harness executes the terminal command. It captures the standard output (`stdout`) and standard error (`stderr`). 
It then sends ONLY the result back to the LLM: *"Your command outputted this."*

### Why this is an Immune System:
*   **No File Collision:** The LLM cannot accidentally overwrite `transcript.md` with `Set-Content` (like I did), because the LLM is not the entity writing the ledger. The Harness does the writing.
*   **No Forgetting:** The LLM cannot "forget" to trail its thoughts when it gets distracted, because the Harness physically forces the `internal_reasoning` field to be populated before any action can be executed.
*   **No Revisionism:** If the LLM makes a mistake, it cannot go back and edit its previous `"internal_reasoning"` to make itself look smarter. The Harness has already sealed that block in the ledger.

This architecture reduces the LLM from a "pseudo-human agent" down to an isolated logic processor. The Harness captures the truth; the LLM merely computes the model is only permitted to compute.
